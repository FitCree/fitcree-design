# FitCree アクセシビリティ実装ガイドライン

## 1. 基本方針

FitCreeはすべてのクリエイターとクライアントが等しく利用できるよう、以下の基準を遵守する。
- 必須 (Must): WCAG 2.1 レベルA（操作不能な箇所の排除）
- 目標 (Should): WCAG 2.1 レベルAA（視認性・コントラストの確保）

## 2. HTMLセマンティクス (Level A)
AIはコード生成時、以下のタグ選択を最優先すること。

- ボタン: クリック可能な要素は `<div>` ではなく必ず `<button>` を使用する。
- テキスト: テキストなどの段落要素には `<div>` ではなく `<p>` を使用する。
- リンク: ページ遷移は `<a>` を使用する。
- 見出し: 視覚的な大きさではなく、情報の階層（`h1` > `h2` > `h3`）に基づいてタグを選ぶ。
- 入力項目: 全てのフォーム要素に `<label>` を付与し、`htmlFor` と `id` で紐付ける。

## 3. インタラクション (Level A & AA)
- キーボード操作: マウスなしですべての操作が可能であること。
- フォーカス可視化 (AA):
  - `outline-none` でフォーカスを消さない。
  - 必ず `focus-visible:ring-2` 等で、現在の選択位置を明示する。
- フォームエラー:
  - エラー時は `aria-invalid="true"` を付与し、エラーメッセージと `aria-describedby`で紐付ける。

## 4. 色と視覚表現 (Level AA)
- コントラスト比:
  - 通常テキストと背景の比率は $4.5:1$ 以上。
  - 太字や大きなテキスト（18pt以上）は $3:1$ 以上。
  - UI要素（ボタンの境界線など）は $3:1$ 以上
- 情報の多重化:
  - 「色」だけで状態を伝えない。エラーは「赤色 + 警告アイコン + テキスト」で表現する。

## 5. スクリーンリーダー対応 (Level A)
- 代替テキスト: `<img>` には必ず `alt` を設定する。装飾目的の場合は `alt=""` とする。
- ARIA属性:
  - アイコンのみのボタンには `aria-label` で説明を付ける。
  - 動的なコンテンツの変化（トースト通知等）には `aria-live` を使用する。
  - 開閉を伴うUI（アコーディオン・ドロップダウン等）には `aria-expanded` を付与する。

## 6. AIへの実装チェックリスト
コード生成・レビュー時に以下を確認すること。

- [ ] クリック可能な要素はすべて `<button>` または `<a>` を使用しているか
- [ ] 全フォーム入力に `<label>` と `htmlFor` / `id` の紐付けがあるか
- [ ] `outline-none` 単独使用がなく、`focus-visible:ring-2` で代替しているか
- [ ] アイコンのみのボタンに `aria-label` があるか
- [ ] エラー状態に `aria-invalid` と `aria-describedby` があるか
- [ ] `<img>` の `alt` が設定されているか（装飾なら `alt=""`）
- [ ] テキストカラーのコントラスト比が基準を満たしているか（`text-neutral-400` 以下は使わない）
