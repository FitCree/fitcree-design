# FitCree デザインシステム & ビジュアル言語
このドキュメントは、FitCreeにおける「正解のデザイン」を定義する聖典です。AIはコード生成時、Figmaを参照する代わりにこのルールを厳格に適用してください。

## 1. デザインコンセプト
FitCreeは、クリエイターの「想い」とクライアントの「ニーズ」を繋ぐプラットフォームです。
- キーワード: 信頼感、情熱、透明性、プロフェッショナル
- 視覚的方針: 基本はクリーンな白・グレーベース。ユーザー属性（Creator/Client）に応じてアクセントカラーを切り替え、モードを明示します。

## 2. カラーシステム (Tailwind CSS Tokens)
AIは、globals.css で定義された変数、または以下のTailwindクラスのみを使用してください。**コンポーネント（.tsx）ファイル内での任意の値（#hex）の直接入力は禁止**します。
※ `globals.css` 内のユーティリティクラス定義（`.link-blue` 等）でのHex使用は許可。ただし新規追加時はCSS変数化を推奨。

### 2.1 ロール別プライマリーカラー
ユーザーのモードによって、メインで使用する色を切り替えます。
| 役割 | Tailwind クラス | Hex | 用途 |
|------|----------------|-----|------|
| Creator | `orange-500` | #F97316 | クリエイターモードの主色。情熱。 |
| Client | `blue-500` | #3B82F6 | クライアントモードの主色。信頼。 |

### 2.2 セマンティック・カラー (ニュートラル)
テキストや背景など、情報の階層構造に使用します。
| カテゴリ | Tailwind クラス | 用途 |
|------|----------------|-----|
| 基本背景 | bg-slate-100 | フォーム画面などのベース背景 |
| カード背景 | bg-white | コンテンツエリア、入力セクション |
| 見出し/本文 | text-neutral-800 | 視認性重視のメインテキスト |
| 補足テキスト | text-neutral-500 | ラベル、キャプション、注釈。**これより薄い（text-neutral-400 以下）は禁止** |
| 境界線 | border-neutral-300 | セクション区切り、入力フォーム枠 |

### 2.3 ステート・カラー (状態)
| 状態 | Tailwind クラス | 備考 |
|------|----------------|------|
| エラー | red-600 | 警告アイコンとセットで使用 |
| 成功 | green-500 | 完了通知、チェックマーク |
| 情報 | sky-500 | TipsBox、ガイドライン用 |

## 3. タイポグラフィ
- **フォント**: `Noto Sans JP` を基本フォントとして使用（`globals.css` で定義）。フォールバックとして `Helvetica Neue`, `Helvetica`, `Arial`, `メイリオ`, `Meiryo`, `sans-serif` を設定。
- **階層ルール**:
  - `h1`: `text-3xl font-bold tracking-tight`
  - `h2`: `text-xl font-semibold`
  - `Body`: `text-base` (16px) 読ませる文章要素（説明文・本文・段落）は原則このサイズ。特別な理由がなければ小さくしない。
  - `Small`: `text-sm` (14px) ラベル・キャプション・注釈の標準サイズ。
  - `XSmall`: `text-xs` (12px) スペースが極めて限られる場合のみ使用可。
- 見出しには `text-neutral-800` を使用し、本文とのコントラストを明確にすること。

### 3.1 テキストカラーとサイズの組み合わせルール

**補足・注釈テキストのデフォルト**: `text-sm text-neutral-500`

| サイズ | 最低カラー | 備考 |
|--------|-----------|------|
| `text-base` (16px) | `text-neutral-600` | 読ませる文章は原則 text-base。さらに薄くしない。 |
| `text-sm` (14px) | `text-neutral-500` | ラベル・注釈の標準。`text-neutral-400` は使用禁止。 |
| `text-xs` (12px) | `text-neutral-500` | スペース制約がある場合のみ許可。それでも `text-neutral-500` を下回ってはならない。 |

**禁止**: `text-neutral-400` 以下（`text-neutral-400`, `text-neutral-300` 等）をUIテキストに使用すること。コントラスト比が不足し可読性が損なわれる。

- 最小フォントサイズは `text-xs` (12px)。`text-[10px]` のような任意数値指定は禁止。
- よほどの理由がない限り `text-sm` を下限とし、`text-xs` は例外扱い。

## 4. スペーシング & レイアウト（重要）
- **セクション間余白**: `p-6` (24px) または `p-8` (32px) を標準とする。
- **基本グリッド**: 原則として **8pxの倍数**（Tailwindの `2, 4, 6, 8, 10...`）を使用すること。
- **微調整**: 8pxでは広すぎる場合のみ、**4pxの倍数**（Tailwindの `1, 3, 5...`）の使用を許可する。
- **禁止**: 奇数ピクセルや、グリッドに沿わない数値の直接指定は禁止。

## 5. 装飾ルール（シャドウ・角丸）

### 5.1 シャドウ
通常のレイアウト（セクション分割、カード配置等）では **shadowを使用せず**、背景色のコントラスト（`bg-slate-100` と `bg-white`）や細いボーダー（`border-neutral-200`）で階層を表現する。

**shadowを使用してよいケース（許可リスト）:**
| 用途 | 許可クラス | 例 |
|------|-----------|-----|
| ドロップダウン・ポップオーバー | `shadow-xl`, `shadow-lg` | ヘッダーメニュー、モーダル |
| モーダル・ダイアログ | `shadow-2xl` | 確認ダイアログ、拡大表示 |
| フローティングボタン（CTA） | `shadow-md`, `shadow-lg` | メインアクションボタン |
| トグルスイッチのつまみ | `shadow` | ToggleSwitch内部 |

上記以外での shadow 使用は禁止。迷った場合はボーダーで代替する。

### 5.2 角丸
| 用途 | クラス | 備考 |
|------|--------|------|
| 標準（カード・セクション・入力） | `rounded-lg` (8px) | デフォルト。迷ったらこれ |
| 大きめのカード・画像コンテナ | `rounded-xl` (12px) | 作品カード、プレビュー領域 |
| モーダル・ダイアログ | `rounded-2xl` (16px) | オーバーレイ系 |
| ピル型（ボタン・バッジ・アバター） | `rounded-full` | タグ、丸ボタン、プロフィール画像 |

`rounded-3xl` は原則使用しない。`rounded-md` 以下も原則不要。

## 6. コンポーネントの実装ルール
AIは新しいUIを作る際、以下の優先順位を守ってください。

1. **既存パーツの再利用**: `fitcree-design-app/src/components/` 内の既存コンポーネントを最優先で使用。
2. **既存パーツが存在しない場合**: `fitcree-design-app/src/components/` 内にコンポーネント作成を優先し、その後ページの組み立て。
3. **外部UIライブラリ**: 現在は未使用。Tailwind + 自前コンポーネントで構築する。

### 6.1 ロール別テーマ切り替え（form-theme.ts）
フォームやインタラクティブ要素では、ユーザーのロール（Creator / Client）に応じてアクセントカラーを切り替える仕組みがある。
- **定義ファイル**: `fitcree-design-app/src/components/forms/elements/form-theme.ts`
- **使い方**: `getFormTheme('creator')` または `getFormTheme('client')` でテーマオブジェクトを取得し、各プロパティ（`ring`, `border`, `bgSelected`, `accent` 等）をクラス名として適用する。
- **新しいフォーム要素を作る際は必ずこの仕組みを利用すること。** ロール別の色をハードコードしてはならない。

## 7. アクセシビリティの厳守
AIは常に docs/accessibility.md を裏側で参照し、以下の数値を担保してください。
- コントラスト比:
  - 通常テキストと背景: $4.5:1$ 以上
  - 大きなテキスト（18pt以上）: $3:1$ 以上
- インタラクション:
  - フォーカス時は必ず focus-visible:ring-2 と ring-offset-2 を付与し、視覚的に強調すること。
  - アイコンのみのボタンには必ず aria-label を付与すること。

## 8. ダークモード方針
現在FitCreeは**ライトモードのみ対応**。`dark:` プレフィックスの使用は禁止する。
`globals.css` 内の `@media (prefers-color-scheme: dark)` ブロックもライトモードと同じ値を設定しており、意図的にダークモードを無効化している。
将来的にダークモード対応を行う場合は、このドキュメントを更新した上で実装すること。

## 9. AIへの特別指示（実装時の思考プロセス）
1. ドキュメントを確認せよ: 実装前に docs/ 内の各定義ファイル（overview.md, design-system.md, architecture.md, accessibility.md）を読み、項目の過不足がないか確認すること。
2. 標準クラスを優先せよ: orange-500 や blue-500 など、定義されたセマンティックなTailwindクラスを使用せよ。
3. レスポンシブを忘れるな: モバイルファーストで設計し、md: 以上のプレフィックスでPCレイアウトを最適化せよ。
